`questions.js` 파일을 읽고, 각 퀴즈 문제의 **정확성**을 검증해줘.

## 검증 항목

CLAUDE.md의 "퀴즈 문제 교차 검증 가이드라인" 4개 항목을 기준으로 각 문제를 점검:

1. **정답 유일성**: `correctAnswer`가 가리키는 선택지만이 유일한 정답인가? 다른 선택지도 해석에 따라 정답이 될 수 있다면 표시.
2. **최상급 표현의 기준**: '가장', '최초', '최대' 등 최상급 표현에 측정 기준(면적·인구·해발 등)이 명시되어 있는가?
3. **시간·범위 명확성**: 시점에 따라 변할 수 있는 정보(현 수도, 최대 도시 등)에 기준 시점이 있는가? 지리적·분류적 범위가 한정되어 있는가?
4. **사실 정확성**: `question` 본문, `options`, `correctAnswer`, `explanation`에 포함된 모든 사실(연도·인명·수치·지명)이 정확한가? 의심스러운 정보는 두 가지 이상 출처로 교차 검증.

## 작업 순서

1. `questions.js`를 읽어 전체 문제를 파악.
2. 각 문제를 위 4개 항목으로 점검. 명백히 정확하고 모호함 없는 문제는 건너뛰어도 됨.
3. 사실 확인이 필요한 항목은 **WebSearch**로 교차 검증 (특히 연도·수치·"가장 ~한" 류 주장).
4. 문제가 발견된 항목만 아래 형식으로 출력.

## 출력 형식

발견된 각 이슈를 아래 형식으로 나열:

```
[id] (카테고리) - <문제 유형: 정답 모호 / 기준 누락 / 시점 누락 / 사실 오류 / 해설 오류>
   질문: <question 본문>
   현재 정답: options[correctAnswer] = "<값>"
   이슈: <구체적 문제점>
   근거: <확인된 사실 또는 참고 출처>
   제안: <수정안 — 질문 보강, 정답 변경, 선택지 수정 등>
```

마지막에 다음을 표시:
- 총 검토 문제 수
- 이슈 발견 건수 (유형별 집계)
- 카테고리별 이슈 분포
- 우선 수정 권장 문제 ID 목록 (사실 오류 > 정답 모호 > 기준 누락 순)

## 참고

- 이슈가 없는 문제는 출력하지 말 것. 검증 통과 메시지로 충분히 간결하게.
- 사실 확인 시 위키백과 단일 출처에 의존하지 말고, 가능하면 공식 기관·백과사전 등 2개 이상 출처로 교차 검증.
- 정답 자체는 맞지만 `explanation`의 부수 정보(연도·수치 등)에 오류가 있다면 "해설 오류"로 별도 표시.
- 모호한 판정은 단정하지 말고 "추가 확인 필요" 표시.
