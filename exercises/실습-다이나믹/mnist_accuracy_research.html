MNIST 손글씨 숫자 인식 정확도 향상 기법 조사 보고서

작성일: 2026-06-29 · 대상 베이스라인: PyTorch Conv-Conv-Pool-FC-FC (~99%) · 목표: 99.5% 이상, 가능하면 99.8%+ (SOTA 영역)

요약 (Executive Summary)

MNIST는 이미 포화된 벤치마크로, 현재 공개된 최고 정확도는 약 99.87~99.91% 수준이며 (단일 모델 99.83%, 앙상블 99.87~99.91%) [1][2][5][6] 0.1% 오류율을 깬 결과(MNIST-NET10, 99.9%)도 보고되어 있습니다. [3][4] 사용자의 ~99% 베이스라인에서 99.5%+로 올리는 가장 비용 효율적인 경로는 (1) affine + elastic distortion 기반 데이터 증강, (2) Batch Normalization + Dropout, (3) cosine annealing 학습률 스케줄이며, 이것만으로도 단일 CNN이 99.6~99.8% 범위에 진입합니다. [10][11][14][16] 99.8%+ SOTA 영역에 진입하려면 다중 모델 앙상블(majority voting, STV) 또는 Homogeneous Vector Capsule / Max-Min CapsNet 같은 캡슐 변형 아키텍처와 TTA를 결합해야 하며, 이 단계의 한계 이득은 +0.03~0.1%p 수준으로 급격히 줄어듭니다. [1][5][7][8][12][15][17] ViT 등 트랜스포머 계열은 MNIST 같은 작은 데이터셋에서 강력한 사전학습 없이는 CNN을 넘기 어려우므로, 본 보고서에서는 CNN/Capsule 계열 + 강한 정규화 + 앙상블을 권장합니다.

1. 현재 MNIST SOTA 상황

MNIST는 사람의 오류율(약 0.2%)에 이미 도달했으며, 0.1~0.2% 오류율 부근에서 모델 차이는 사실상 통계적 잡음과 구분이 어렵습니다. [11]

모델 / 기법 정확도 (오류율) 핵심 아이디어 출처
MNIST-NET10 (Tabik 외, 2020) 99.90% (오류 10/10,000) 확신도(degree-of-certainty) 기반 이종 딥넷 융합 [3][4]
2-layer 이종 앙상블 (An 외, 2020) 99.91% 3x3 / 5x5 / 7x7 커널 단순 CNN 세 동종 앙상블의 이종 결합 [1]
3-모델 majority voting (An 외, 2020) 99.87% 커널 크기만 다른 3개 단순 CNN의 다수결 [2]
Branching/Merging CNN + HVC 앙상블 (Byerly 외, 2020) 99.87% Homogeneous Vector Capsule, no routing [5][12]
Branching/Merging CNN + HVC 단일 99.83% 단일 모델 SOTA [6][12]
3-모델 Max-Min CapsNet 앙상블 99.83% (오류 0.17%) Max-Min 정규화 캡슐 + majority vote [8]
단일 Max-Min CapsNet 99.80% (오류 0.20%) routing 정규화 개선 [7]
MCDNN (Cireşan 외, 2012) 99.77% (오류 0.23%) 35-컬럼 multi-column DNN + epoch마다 distortion [11][13]

2. 데이터 증강 (Data Augmentation)

"MNIST에서 표준적인 input distortion을 적용하면 오류율이 더 줄어든다"는 것은 textbook 수준의 합의입니다. [9] 핵심 증강은 affine 변환 + elastic deformation이며, 이 두 가지가 MCDNN을 99.77%까지 끌어올린 핵심 동력이었습니다. [11][13]

2.1 사용자에게 권장되는 증강 (강한 효과 → 약한 효과 순)

기법대략적 이득구현 난이도비고
Affine (회전 ±10~15°, 이동 ±2px, 스케일 ±10%, shear) +0.3 ~ +0.6%p 낮음 torchvision.transforms.RandomAffine 한 줄. 단독으로도 99.6%+ 가능 [14]
Elastic distortion (Simard 2003) +0.2 ~ +0.4%p (affine 위에 추가) 중간 MCDNN, MNIST-NET10 등 SOTA 모델이 공통적으로 사용. torchvision.transforms.ElasticTransform 또는 albumentations [11][13]
매 epoch마다 새 distortion 샘플링 고정 증강 대비 추가 이득 낮음 DataLoader가 자동으로 처리. MCDNN의 핵심 디테일 [13]
RandAugment / AutoAugment +0.05 ~ +0.15%p (한계 이득) 낮음 이미 affine+elastic이 있으면 추가 이득 작음. ImageNet 수준에서는 강력
Mixup / Cutout / CutMix +0.0 ~ +0.1%p 낮음 MNIST에서는 효과 미미. Cutout은 오히려 작은 28x28 이미지에서 부정적일 수 있음
주의: MNIST는 좌우 반전이 의미적으로 다른 숫자가 되므로 RandomHorizontalFlip은 사용하지 마세요 (3 ↔ E 등). 회전도 ±15°를 크게 초과하면 6↔9 혼동을 유발합니다.

3. 모델 아키텍처 개선

3.1 무엇이 효과적인가

  • 커널 다양화 + Batch Normalization 기반 단순 CNN: An 외(2020)는 BN+ReLU만 들어간 단순 CNN 3개(커널 3/5/7)로 99.87%를 달성했으며, 이는 ResNet 같은 깊은 구조 없이도 SOTA에 도달함을 보여줍니다. [1][2]
  • Branching/Merging + Homogeneous Vector Capsules: routing 없는 캡슐 + 멀티 분류 분기 헤드 구조로 단일 모델 99.83%, 앙상블 99.87%. [5][6][12]
  • Max-Min CapsNet: 캡슐 routing 활성치를 Max-Min 정규화로 안정화하여 단일 99.80%, 3-모델 앙상블 99.83%. [7][8]
  • Multi-column DNN (역사적 SOTA): 35개의 GPU 트레이닝된 컬럼을 averaging하여 99.77% 달성, 인간 오류율(0.2%)에 도달한 최초 사례. [11]

3.2 효과가 제한적이거나 권장되지 않는 것

  • Vision Transformer (ViT): 대규모 사전학습 없이 작은 데이터셋(60K 샘플)으로 ViT를 학습하면 동일 파라미터의 CNN보다 잘 나오기 어렵습니다. MNIST에서 ViT가 의미 있게 우위를 가진 결과는 보고된 적이 없습니다.
  • 매우 깊은 ResNet: 28x28 흑백 이미지에는 과한 capacity로, ResNet-18~50 정도가 한계이며 단순 BN-CNN 대비 명확한 이득은 없습니다.

4. 정규화 및 학습 기법

기법대략적 이득 (베이스라인 위에)구현 난이도근거 / 비고
Batch Normalization (모든 Conv/FC) +0.2 ~ +0.5%p, 수렴 안정화 낮음 hwalsuklee 레퍼런스: BN + Dropout(FC3) + augmentation으로 99.61% 달성 [14]
Dropout (FC 또는 마지막 Conv 뒤, p=0.25~0.5) +0.1 ~ +0.3%p 낮음 강한 augmentation과 함께 쓰면 한계 이득이 줄어듦 [14]
Label smoothing (ε=0.05~0.1) +0.05 ~ +0.2%p 낮음 He 외(2019) "Bag of Tricks": 다른 기법과 결합 시 누적 효과 (ResNet-50에서 75.3→79.29% 등) [9]
Cosine LR + Warm Restarts (SGDR) +0.1 ~ +0.3%p, 수렴 가속 낮음 Loshchilov & Hutter (2017). torch.optim.lr_scheduler.CosineAnnealingWarmRestarts [10]
Optimizer 선택: SGD + Nesterov momentum (0.9) + weight decay (5e-4) 일반화 측면에서 Adam 대비 +0.05 ~ +0.2%p 낮음 MNIST 분류에서는 SGD+momentum이 최종 정확도에서 약간 더 좋은 경향. Adam은 수렴은 빠르나 일반화 약간 손해
Knowledge distillation +0.05 ~ +0.15%p 중간 Bag of Tricks 조합에 포함됨 [9]
Weight decay / L2 +0.05 ~ +0.1%p 낮음 5e-4 ~ 1e-3 범위 권장
"Bag of Tricks for Image Classification"에서 label smoothing + cosine LR + mixup + knowledge distillation을 모두 결합하면 ResNet-50의 ImageNet top-1 정확도가 75.3% → 79.29%로 약 4%p 향상되었습니다. [9] MNIST에서는 절대 이득은 더 작지만(이미 99% 부근), 누적 효과로 99.5% → 99.7~99.8% 진입에 충분합니다.

5. 앙상블 (Ensembling), TTA, Pseudo-labeling

5.1 앙상블 — 99.5%→99.8% 진입의 가장 확실한 지렛대

앙상블 방식관찰 결과구현 난이도출처
5-모델 majority voting 단일 99.61% → 앙상블 99.72% (+0.11%p) 낮음 [15]
15-모델 평균 앙상블 최대 99.79%, 평균 99.745% (100회 실행, σ=0.02) — 추가 모델 ↑ 한계 이득 감소 중간 (학습 비용) [16]
3-모델 majority vote (이종 커널) 99.87% — 동일 아키텍처 3x3/5x5/7x7 커널 다양화 중간 [2]
2-layer 이종 앙상블 (앙상블의 앙상블) 99.91% — MNIST 최상위 결과 중 하나 높음 [1]
Single Transferable Vote (STV) 실험적으로 majority voting과 거의 동일 (99.36% vs 99.35%) — 큰 차이 없음 중간 [18]
Degree-of-certainty 융합 (MNIST-NET10) 이종 네트워크 + 데이터 + 융합 전략 모두 다양화 → 99.9% (오류 10개) 높음 [3][4]

5.2 Test-Time Augmentation (TTA)

TTA는 테스트 이미지 각각에 대해 여러 번의 증강(회전, 이동 등) 복사본을 만들고 모델 예측을 평균/투표로 합치는 기법입니다. [17] MNIST에서는 보통 +0.05 ~ +0.15%p의 한계 이득을 얻을 수 있으며, 추가 학습 없이 추론 비용만으로 정확도를 끌어올리는 가장 저렴한 방법 중 하나입니다.

  • 구현: 동일 augmentation pipeline을 test 단계에서도 5~20회 적용 → 각 예측 평균
  • 난이도: 낮음
  • 훈련용 증강과 동일한 분포의 변환을 사용하는 것이 핵심

5.3 Pseudo-labeling

MNIST는 라벨이 모두 주어진 supervised 데이터셋이므로 외부 unlabeled 데이터(예: USPS, EMNIST의 digit subset)를 끌어와야 의미가 있습니다. 표준 MNIST 평가에서는 일반적으로 사용하지 않으며, 본 보고서의 범위 밖입니다.

6. 권장 로드맵 (사용자 베이스라인 ~99% 기준)

1단계: 99% → 99.5%+ (낮은 난이도, 즉시 적용)

  1. 모든 Conv/FC 레이어 뒤 Batch Normalization 추가 [14]
  2. RandomAffine(rotation=±10~15°, translate=±2px, scale=0.9~1.1) 적용 [14]
  3. FC 직전 Dropout(p=0.5) [14]
  4. Optimizer: SGD(lr=0.01, momentum=0.9, nesterov=True, weight_decay=5e-4)
  5. 스케줄: CosineAnnealingWarmRestarts(T_0=10), 80~100 epoch [10]
  6. 예상 결과: 99.5 ~ 99.65%

2단계: 99.5% → 99.75% (중간 난이도)

  1. 위에 Elastic distortion 추가 (alpha=8, sigma=3 부근) [11][13]
  2. Label smoothing (ε=0.1) [9]
  3. 매 epoch마다 distortion 새로 샘플링 [13]
  4. TTA 5~10회 (rotation/shift 변형) [17]
  5. 예상 결과: 99.70 ~ 99.80%

3단계: 99.75% → 99.85%+ (높은 난이도, SOTA 영역)

  1. 커널 크기를 다르게 한 단순 CNN 3~5개 학습 (3x3 / 5x5 / 7x7) [1][2]
  2. Majority voting 또는 confidence-weighted ensemble [1][2][15][16]
  3. (선택) Branching/Merging + Homogeneous Vector Capsules 도입 [5][6][12]
  4. (선택) 2-layer 이종 앙상블로 모델군 자체를 앙상블 [1]
  5. 예상 결과: 99.83 ~ 99.91% (현재 공개 SOTA 영역)

7. 한계 및 주의사항 (Caveats)

  • 99.8% 이상에서는 통계적 잡음과 실질 개선이 거의 구분 불가능합니다. 15-모델 앙상블 100회 실행 결과의 표준편차가 0.02%p에 불과합니다. [16] 단일 run으로 99.81%가 나와도 다음 run에서 99.70%일 수 있습니다.
  • 테스트셋 누설(leakage)에 주의: 어떤 augmentation/하이퍼파라미터도 validation/test에서 직접 튜닝하지 마세요. MNIST 테스트셋은 10,000개에 불과해 작은 차이가 과적합되기 쉽습니다.
  • 인간 오류율(~0.2%)이 사실상 상한이며, 0.1% 이하는 사람조차 일관되게 정답을 매기기 힘든 모호한 글자들이 포함되어 있습니다. [11]
  • STV 같은 정교한 투표 알고리즘은 majority voting 대비 의미 있는 이득을 보여주지 못했습니다(99.36% vs 99.35%). [18] 단순 다수결로 충분합니다.
  • 새 시도가 더 좋은지 평가하려면 여러 random seed로 반복 실행 후 평균 ± 표준편차로 비교해야 합니다. [16]

8. 남은 질문 (Open Questions)

  • 현대 ViT 변형(Swin, DeiT, ConvNeXt 등)을 MNIST에서 처음부터 학습했을 때 99.8%+에 도달한 공개 결과는 거의 없으며, 정량적 비교 데이터가 부족합니다.
  • EMNIST나 USPS 같은 외부 데이터를 활용한 semi-supervised / pseudo-labeling으로 MNIST에서 99.91%를 넘은 사례는 본 조사 범위에서 확인되지 않았습니다.
  • HVC 캡슐 + 강한 distillation의 조합이 99.91%를 넘을 수 있는지에 대한 결정적 결과는 아직 공개되지 않았습니다.
