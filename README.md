### 02. 필수 명령어 실습

#### 터미널 기본 명령어

| 명령어 | 설명 |
|---|---|
| `claude` | 클로드 코드 실행 |
| `claude "프롬프트"` | 프롬프트와 함께 시작 |
| `claude -c` | 가장 최근 대화 이어서 계속 |
| `claude update` | 최신 버전 업데이트 |
| `claude --model <모델명>` | 모델 선택 |
| `claude --debug` | 디버그 모드 |

**선택 가능한 모델**

| 모델 | 특징 |
|---|---|
| `fable` | 가장 강력 |
| `opus` | 강력 |
| `sonnet` | 균형 |
| `haiku` | 빠름 |

#### 세션 관리 명령어

> 세션: 사용자와 시스템이 연결되어 있는 일정한 기간

| 명령어 | 설명 |
|---|---|
| `/clear` | 대화 내용을 모두 지우고 새로 시작 |
| `/rewind` | 이전 대화 상태로 되돌리기 |
| `/compact` | 대화 내용을 압축해서 정리 |
| `/tasks` | 백그라운드 작업 목록 확인 및 관리 |

#### 정보 확인 명령어

| 명령어 | 설명 |
|---|---|
| `/help` | 사용 가능한 명령어 목록 (Tab 키로 카테고리 전환) |
| `/status` | 모델, 버전, 계정 등 현재 상태 |
| `/usage` | 플랜 한도와 사용 현황 (5시간/주간) |
| `/context` | 현재 대화의 컨텍스트 사용량 |

> 컨텍스트: 현재 대화와 작업 정보를 저장하는 메모리 공간

#### 설정 및 구성 명령어

| 명령어 | 설명 |
|---|---|
| `/config` | 설정 화면 |
| `/model` | AI 모델 전환 |
| `/init` | 프로젝트에 CLAUDE.md 생성 |
| `/add-dir` | 추가 작업 디렉토리 지정 |
| `/permissions` | 접근 권한 관리 |
| `/terminal-setup` | 터미널 설정 (Shift+Enter 줄바꿈 등) |

#### 작업 모드 명령어

| 명령어 | 설명 |
|---|---|
| `/plan` | 플랜 모드 (코드 수정 없이 계획 수립) |

#### 프로젝트 작업 명령어

| 명령어 | 설명 |
|---|---|
| `/plugin` | 플러그인 설치 및 관리 |
| `/memory` | CLAUDE.md 메모리 파일 편집 |

> 플러그인: 커스텀 커맨드, MCP 설정, 서브에이전트 등을 패키지로 묶어 명령어 하나로 설치/공유

#### 문제 해결 명령어

| 명령어 | 설명 |
|---|---|
| `/doctor` | 설치 상태 진단 |
| `/bug` | 버그 보고 |

#### MCP 기본 명령어

| 명령어 | 설명 |
|---|---|
| `/mcp` | MCP 서버 연결 상태 확인 |
| `claude mcp list` | 설정된 MCP 서버 목록 |
| `claude mcp add` | MCP 추가 |
| `@<서버>:<리소스>` | MCP 리소스를 대화에 불러오기 (예: `@github:issue://123`) |

**MCP 추가 명령**

```bash
# 원격 (HTTP)
claude mcp add --transport http <내가 정한 이름> <서버 URL>

# 로컬
claude mcp add <내가 정한 이름> <옵션> -- <MCP 실행 명령>
```

## Part 4. CLAUDE.md와 커스텀 명령어

### 01. CLAUDE.md 파일 만들고 활용하기

```
/init
```

### 02. 프로젝트 설정 파일 활용하기

**[샘플 예제]** [Study-03-basic 다운로드](https://download-directory.github.io/?url=https://github.com/taehojo/vibecoding/tree/master/Study-03/Study-03-basic)

**퀴즈 문제 교차 검증 가이드라인** (CLAUDE.md에 추가)

```
퀴즈 문제 교차 검증 가이드라인

모든 문제 작성 시 확인 사항
1. 정답이 하나뿐인가?
   - 다른 해석 가능 시 조건 명시 (예: 면적 기준, 2024년 기준)
2. 최상급 표현에 기준이 있는가?
   - '가장 큰', '최초의' 등 표현에 측정 기준 명시
3. 시간과 범위가 명확한가?
   - 변할 수 있는 정보는 시점 명시
   - 지리적, 분류적 범위 한정
4. 교차 검증했는가?
   - 의심스러운 정보는 2개 이상 출처 확인
```

### 03. 커스텀 명령어 만들기

**실습 프롬프트**

```
.claude/commands 폴더를 생성하고 quiz-validate.md 파일을 만들어서 퀴즈 문제 중에서 '가장', '최초', '최대' 같은 최상급 표현이 있는지 찾아서 목록으로 보여주게 해줘.
```

### 04. 나만의 자동 검증 시스템 만들기

**커스텀 명령어 생성**

```
.claude/commands/quiz-check.md를 생성해서 문제의 정확성을 검증하게 해줘.
.claude/commands/quiz-range.md를 생성해서 문제의 난이도와 정답 분포를 확인하게 해 줘.
```

**검증 실행**

```
/quiz-check로 정확성을 검증하고 /quiz-range로 난이도와 정답 분포를 확인한 후 결과를 보여줘.
```

## Part 5. OpenRouter API 활용

### 01. 오픈 라우터 검색과 API 키

1. [OpenRouter](https://openrouter.ai) 가입
2. API 키 발급
3. 프로젝트 폴더에 `.env` 파일 생성 후 저장

**실습 프롬프트**

```
OpenRouter에서 받은 API 키를 .env 파일로 저장했어. 이 키를 안전하게 사용할 수 있도록 준비해 줘.
```

### 02. 다양한 AI 모델의 API 활용하기

> `[자신이 선택한 모델명]` 부분은 본인이 고른 모델로 바꿔서 입력하세요.

**실습 프롬프트**

```
이제 준비된 API가 실제로 작동하는지 테스트해 줘.
이미지 인식은 [자신이 선택한 모델명] 모델, 텍스트 인식은 [자신이 선택한 모델명] 모델을 이용해줘.
```

### 03. 스스로 생각하고 판단하는 앱

**실습 프롬프트**

```
앞서 만든 OpenRouter API를 이용해서, 냉장고 사진에서 재료를 인식하고 레시피를 추천하는 웹 애플리케이션을 만들고 싶어. 먼저, 이미지를 입력받고 [자신이 선택한 텍스트 모델명]을 사용해서 이미지를 인식한 다음, [자신이 선택한 텍스트 모델명]을 사용해서 레시피를 생성하는 PRD를 작성해줘.
```

## Part 6. 서브 에이전트

### 01. 서브 에이전트의 개념과 실행

**코드 리뷰어 에이전트 정의**

```
코드를 읽고 버그는 없는지, 코딩 규칙에 따라 올바르게 작성되었는지를 점검하고 성능 최적화를 제안하는 코드 리뷰어.
```

**실습 프롬프트**

```
[에이전트 이름]에게 '냉장고를 부탁해' 어플리케이션 코드를 검토하게 해줘.
```

### 02. 서브 에이전트간의 협업 기획

**에이전트 역할 정의**

| 역할 | 책임 |
|---|---|
| **백엔드 개발자** | 서버 아키텍처 설계, API 개발, 데이터 처리, 외부 서비스 통합, 보안 및 성능 최적화. 안정적이고 확장 가능한 백엔드 시스템 구축 |
| **프런트엔드 개발자** | 사용자 인터페이스 설계 및 구현, 반응형 디자인, 웹 접근성, 성능 최적화 |
| **품질 보증 엔지니어** | 기능 테스트, 에러 처리 검증, 성능 최적화, 코드 리뷰. 버그 발견 및 사용성 개선 제안 |

### 03. 서브 에이전트를 활용한 실전 앱 제작

**실습 프롬프트**

```
AI 공감 다이어리를 만들어 줘. 오늘 있었던 일을 한 줄로 쓰면, AI가 감정을 분석하고 공감하며 위로해 주는 일기 애플리케이션이야. 최종 버전은 브라우저에서 바로 열 수 있는 'index.html' 파일이어야 해.

먼저 [백엔드 개발자 에이전트명]가 OpenRouter API를 연동해서 감정 분석과 공감 메시지를 생성하는 기능을 구현해 줘.
AI API는 [오픈라우터 모델 이름]을 사용하고, API 키는 현재 폴더의 '.env' 파일에 저장된 것을 사용해.

[프런트엔드 개발자 에이전트명]가 따뜻하고 편안한 느낌의 일기장 UI를 만든 다음, [품질 보증 엔지니어 에이전트명]가 실제로 여러 상황에서 문제없이 작동하는지 테스트해 줘.
```

## Part 7. MCP, 스킬, 플러그인

### 01. MCP의 개념 이해하고 연결하기

1. [Vibe Index](https://www.vibeindex.ai)에서 `notion` 검색
2. MCP 추가:

```bash
claude mcp add --transport http notion https://mcp.notion.com/mcp
```

### 02. 스킬 이해하고 설치하기

1. [Vibe Index](https://www.vibeindex.ai)에서 `vibeindex` 검색
2. 스킬 설치:

```bash
npx vibeindex add vibeindex/skills --skill Vibeindex
```

### 03. 플러그인, 마켓플레이스 활용

1. [Vibe Index](https://www.vibeindex.ai) > 마켓플레이스 메뉴 > `anthropics-skills` 선택
2. 마켓플레이스 추가:

```
/plugin marketplace add anthropics/skills
```
